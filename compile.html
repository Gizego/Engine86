Compiling the Library
This is different for every platform

Windows MinGW:

The engine was made for GCC. With MinGW you should get everything you need.
The libraries in the "dependencies" folder were compiled for windows so you won't have to build any dependency.
The build32.bat and build64.bat files should be slightly altered to match your MinGW/bin folder.


Visual Studio on Windows:

This is untested but you may have to rebuild GLEW and GLFW to MSVC libraries and get them on the "dependencies" folder.
Don't forget to change the suffix of the libraries to ".a" even when they are MSVC lib files.
Make a solution file and statically link the libraries in the "dependencies folder" for your targeted CPU architecture.


Linux and Mac GCC:

In this case, you'll have to rebuild all the libraries for your platform.
You'll extract some important information from the BAT file. It won't work directly on your platfrom but it shows you what you should do with GCC.
Don't forget to change the path of the libraries and remove the -lgdi32 dependency.
